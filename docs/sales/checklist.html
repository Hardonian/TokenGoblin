# TokenGoblin — Sales Completion Checklist Use this before sending any buyer a package, quote, or deployment option. - [ ] Value is stated in buyer terms: spend visibility, leak reduction, routing savings, forecasting, governance. - [ ] No placeholders or demo-only claims. Pricing matches `docs/sales/packages.html`. - [ ] Sales path is clear: SaaS Stripe checkout, self-hosted option, enterprise contact path. - [ ] Buyer risks are named: data, compliance, self-hosted access, support expectations. - [ ] Next action is explicit: signup, trial, demo call, deployment review, or written proposal target date. ## Accepted quote format - Package name + price + billing period - Included limits and support level - Optional add-ons: annual billing, custom onboarding, SLA terms

💡 **What**: Extracted all `regexp.MustCompile` calls in `RefinePrompt` out of the function body and precompiled them at initialization time. Used package-level variables for static regexes and a thread-safe global map (with `sync.RWMutex`) to lazily compile and cache dynamic slop phrase regexes.

🎯 **Why**: Compiling regular expressions repeatedly inside loops or frequently called functions is a known anti-pattern in Go, as `MustCompile` is relatively expensive and blocks the critical path. Moving compilation to initialization time dramatically reduces CPU time and memory allocations per call, increasing overall system throughput for prompt refinement.

📊 **Measured Improvement**:
A benchmark was run with an input string containing conversational slop, multiple spaces, line breaks, and JSON. The results are significant:

*   **Baseline (Before)**: ~251,773 ns/op, 91,240 B/op, 807 allocs/op
*   **Optimized (After)**: ~117,934 ns/op, 7,339 B/op, 100 allocs/op
*   **Result**: CPU time reduced by **~53%** (more than 2x faster). Memory allocations dropped by **~88%** per operation.
